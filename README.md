# Routine Tracker

> **学習 × 筋トレ — 2軸のバランスをデータで最適化する**

日々の活動を記録・可視化し、**「今日何をすべきか」** をデータに基づいて導き出すための意思決定支援アプリです。

---

## 💡 プロダクト背景

私自身、プログラミング学習に没頭すると運動不足に、運動に集中すると学習時間が減る——この2つのジレンマを抱えていました。

毎日のルーティンを「学習」と「筋トレ」の2軸で記録し、GitHubの草のように可視化することで、自分では気づきにくい**行動の偏り**を認識できるようにしたい。この思いから開発しました。

### ターゲットユーザー
- 学習と運動のバランスを管理したい人
- 自分の活動パターンをデータで把握したい人

---

## 🛠 技術スタック

| 技術 | 用途 | 選定理由 |
|------|------|---------|
| **Next.js 16 (App Router)** | フロントエンド + APIルート | 高速な画面遷移と開発効率 |
| **React 19** | UIコンポーネント | 宣言的なUI記述とHooksによる状態管理 |
| **TypeScript** | 言語 | 型安全なドメインモデルとUI。プロジェクトの核 |
| **Tailwind CSS v4** | スタイリング | ユーティリティファーストで動的スタイルを軽量に実装 |
| **Firebase** | 認証 + データベース | Google OAuth + Firestore（現在稼働中） |

### 技術選定のポイント

**Firebase（Auth + Firestore）を最終スタックとして確定**
Clerk + Supabase（PostgreSQL）への移行も検討しましたが、応募・面談フェーズに入るこの時期に動いている実装を作り直すリスクを避け、Firebaseでの完成を選択しました。「より良い技術」より「動いているものを壊さない」期日から逆算したリスク管理の判断です。

## 💎 こだわり — 学習プロセス自体を体系化

このプロジェクトの最大の特徴は**技術力の身につけ方**にあります。

AIアシスタントを「仕事を代わりにやらせる道具」ではなく「最強の教師」として活用するため、6段階の学習プロセスを確立しました：

| STEP | 内容 | ブランチ |
|------|------|---------|
| 1 | AIがコード生成 | `draft/*` |
| 2 | AIが1行ずつ解説 | — |
| 3 | 理解した上で写経 | `feat/handson-*` |
| 4 | 全行に自分で解説コメントを記述 | `feat/handson-*` |
| 5 | AIにコメントの正誤確認 | — |
| 6 | コメントを消して記憶だけで再実装 | `feat/recall-*` |

「動いた」ではなく**「自分の言葉で説明できる」** まで追求しています。
このプロセスにより、面接で「なぜこの書き方にしたんですか？」と聞かれた時に答えられる実力を養っています。

## 📦 開発環境
