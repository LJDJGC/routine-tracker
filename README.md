# Routine Tracker

> **学習 × 筋トレ × 英語イマージョン — 3軸のバランスをデータで最適化する**

日々の活動を記録・可視化し、**「今日何をすべきか」** をデータに基づいて導き出すための意思決定支援アプリです。

---

## 💡 プロダクト背景

私自身、プログラミング学習に没頭すると運動不足に、運動に集中すると学習時間が減り、英語学習も後回しになる——この3つのジレンマを抱えていました。

毎日のルーティンを「精神（学習）・身体（筋トレ）・言語（英語）」の3軸で記録し、GitHubの草のように可視化することで、自分では気づきにくい**行動の偏り**を認識できるようにしたい。この思いから開発しました。

### ターゲットユーザー
- Comprehensible Input / イマージョン学習でYouTube英語動画を視聴している学習者
- 学習・運動・語学の3軸をバランス管理したい人

---

## 🛠 技術スタック

| 技術 | 用途 | 選定理由 |
|------|------|---------|
| **Next.js 16 (App Router)** | フロントエンド + APIルート | 高速な画面遷移と開発効率 |
| **React 19** | UIコンポーネント | 宣言的なUI記述とHooksによる状態管理 |
| **TypeScript** | 言語 | 型安全なドメインモデルとUI。プロジェクトの核 |
| **Tailwind CSS v4** | スタイリング | ユーティリティファーストで動的スタイルを軽量に実装 |
| **Firebase** | 認証 + データベース | Google OAuth + Firestore（現在稼働中） |

### 技術選定のポイント（移行経験）

**Firebase → Clerk + Supabase（PostgreSQL）への移行を計画中**
当初FirebaseでAuth + Firestoreを実装しましたが、よりモダンな構成への移行を判断しました。

- **なぜClerkか**: Next.js App Routerとの統合が最も簡単
- **なぜSupabaseか**: PostgreSQL（RDB）で実務レベルの知識が身につく
- **なぜ移行したか**: NoSQLからRDBへの移行判断自体が、技術選定の判断力を示す経験として価値がある

## 💎 こだわり — 学習プロセス自体を体系化

このプロジェクトの最大の特徴は**技術力の身につけ方**にあります。

AIアシスタントを「仕事を代わりにやらせる道具」ではなく「最強の教師」として活用するため、6段階の学習プロセスを確立しました：

| STEP | 内容 | ブランチ |
|------|------|---------|
| 1 | AIがコード生成 | `draft/*` |
| 2 | AIが1行ずつ解説 | — |
| 3 | 理解した上で写経 | `feat/handson-*` |
| 4 | 全行に自分で解説コメントを記述 | `feat/handson-*` |
| 5 | AIにコメントの正誤確認 | — |
| 6 | コメントを消して記憶だけで再実装 | `feat/recall-*` |

「動いた」ではなく**「自分の言葉で説明できる」** まで追求しています。
このプロセスにより、面接で「なぜこの書き方にしたんですか？」と聞かれた時に答えられる実力を養っています。
